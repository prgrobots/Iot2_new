    Week-by-Week AWS Content (Weeks 10–18)¶

    Phase 1: AWS IoT Foundations (Weeks 10–14)¶

    Week 10: AWS IoT Core & Secure Connection¶

    • Courses:
    • Getting Started with AWS IoT (Foundations, 35 min)
    • Securely Connecting AWS IoT Devices to the Cloud (Foundations, 1 h)
    • Duration: 1.5 hours
    • Type: Free digital courses
    • Why: Core intro + X.509 certificates for haul-truck MCUs
    • Real-hardware extension: Register truck as Thing, flash certificates to ESP32, first MQTT publish

    Week 11: Device Data, Rules & Shadows¶

    • Course: Handling AWS IoT Device Data and States (Foundations, 2 h)
    • Duration: 2 hours
    • Type: Free digital course
    • Why: MQTT, Rules Engine, Device Shadows (offline mining operations)
    • Real-hardware extension: Device Shadow for engine temp + cabin lock state

    Week 12: Fleet Management & OTA Updates¶

    • Courses:
    • Managing AWS IoT Devices at Scale (Foundations, 3.5 h)
    • OTA Updates in Edge Devices (via NMTAFE sandbox, 1 h)
    • Duration: 4.5 hours
    • Type: Free Foundations + NMTAFE sandbox course
    • Why: Fleet indexing, OTA for 400-ton trucks in the Pilbara
    • Real-hardware extension: OTA firmware push to change vibration threshold

    Week 13: Edge Processing & Industrial Protocols¶

    • Courses:
    • ETL at the Edge (via NMTAFE sandbox, 1 h)
    • Lab – Deploying AWS IoT Greengrass Components (Free lab, 1 h)
    • Duration: 2 hours
    • Type: NMTAFE sandbox course + Free lab
    • Why: Local data processing on truck before cloud transmission
    • Real-hardware extension: Deploy Greengrass core on one ESP32, local anomaly detection component

    Week 14: Analytics & Visualization¶

    • Courses:
    • Analyzing, Visualizing, and Gaining Insights from IoT Devices (Foundations, 45 m)
    • Connecting Systems and Machines to AWS for Industrial Manufacturing (Free digital course, 45 m)
    • Duration: 1.5 hours
    • Type: Free digital courses
    • Why: QuickSight + SiteWise for fleet dashboard
    • Real-hardware extension: SiteWise asset model "RockCore-Truck-01" with real data

    Phase 2: AWS Industry Quest (Weeks 15–17)¶

    Weeks 15–17: AWS Industry Quest – Manufacturing and Automotive¶

    • Course: AWS Industry Quest: Manufacturing and Automotive (subscription course, 13 h total split over 3 weeks)
    • Duration: ~4–5 hours per week (13 hours total)
    • Type: NMTAFE sandbox access (you add students)
    • Why: Digital twins, edge-to-cloud, predictive maintenance, vehicle telemetry — literally built for haul trucks
    • Real-hardware extension: Build digital twin of 3D-printed truck in AWS IoT TwinMaker, overlay live sensor data, implement predictive maintenance

    Labs covered in Industry Quest:

    • Lab 1: Introduction to digital twins and AWS IoT TwinMaker
    • Lab 2: Modeling manufacturing equipment (truck example)
    • Lab 3: Connecting real-time data sources
    • Lab 4: Fleet simulation (multiple assets)
    • Lab 5: Predictive maintenance algorithms
    • Lab 6: Edge-to-cloud data pipeline
    • Lab 7–13: Manufacturing scenarios, vehicle telemetry, scaling to production

    Phase 3: Capstone Submission (Week 18)¶

    • No new AWS content – Week 18 is for final demo and portfolio submission
    • Integration: Bring Weeks 1–17 together (electronics + AWS + digital twins)
    • Deliverable: 5-minute system walkthrough video + portfolio document

    Certification Path (Optional)¶

    AWS Certified Cloud Practitioner¶

    • Exam Code: CLF-C02
    • Cost: $100 USD
    • Duration: 90 minutes
    • Study Resources: AWS Skill Builder courses (above)
    • Recommended: After completing Weeks 10-14, students can sit exam
    • Value: Entry-level AWS certification, demonstrates cloud competency

    Student Support¶

    Each student should have:

    • AWS account (free tier with sandbox credits)
    • Enrolled in AWS Skill Builder Foundations courses (weeks 10-14) – you enroll them
    • Added to NMTAFE sandbox for AWS Skill Builder paid courses (weeks 12-13) – you add via NMTAFE admin
    • Added to NMTAFE sandbox for AWS Industry Quest (weeks 15-17) – you add via NMTAFE admin
    • GitHub repository for code submission
    • AWS IoT console access (limited to their Thing)

    No student subscription cost — everything provided via your NMTAFE account.

    Technical Issues:

    • AWS IAM permissions: Contact course instructor
    • NMTAFE sandbox access: Contact NMTAFE admin
    • Device connectivity: Verify IoT Core rule routing

    For technical support with AWS resources or subscription setup, contact course instructors.
