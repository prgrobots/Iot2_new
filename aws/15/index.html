    Session 15: AWS Industry Quest – Manufacturing & Automotive (Week ⅓)¶

    Week: 15
    Element: ICTIOT503 Element 3: Deploy complete IoT solution with digital twin
    Duration: ~4–5 hours
    Phase: Capstone Integration (AWS Industry Quest: Manufacturing & Automotive)

    Session Introduction¶

    This week marks the start of AWS Industry Quest: Manufacturing and Automotive, a game-based immersive learning environment specifically built for vehicle and equipment telemetry scenarios. Over the next 3 weeks, you'll complete this 13-hour course that covers digital twins, edge-to-cloud architectures, predictive maintenance, and vehicle fleet simulation — directly aligned with the RockCore haul truck scenario.

    This week (Week 15): Introduction to the Industry Quest, digital twins in AWS IoT TwinMaker, and building your first digital twin model of the 3D-printed haul truck.

    Learning Objectives¶

    By the end of Week 15, you will be able to:

    • Understand AWS IoT TwinMaker and digital twin concepts
    • Build a digital model of your 3D-printed truck in TwinMaker
    • Connect real IoT sensor data to the digital twin
    • Visualize live data from physical truck on the digital model
    • Prepare for fleet simulation and predictive maintenance labs

    Session Structure¶

    1. AWS Industry Quest: Manufacturing and Automotive – Labs 1–4 (3–4 h this week)
    2. Digital Twin Modeling – Create RockCore-Truck-01 in TwinMaker
    3. Sensor Data Binding – Connect MQTT telemetry to digital twin properties
    4. Real-Hardware Extension – Capture live data on digital model

    Pre-Session Preparation¶

    Required Completion

    • AWS account with Industry Quest: Manufacturing and Automotive enrollment (you added students to subscription)
    • Haul truck device from Weeks 1–14 publishing to AWS IoT Core
    • SiteWise asset model "RockCore-Truck-01" from Week 14 (optional but recommended)
    • AWS IoT TwinMaker enabled in your region (N. Virginia recommended)

    Hardware Check

    • Real ESP32 haul truck publishing MQTT telemetry
    • Sensors functional: temperature, vibration, gas, cabin lock
    • AWS IoT console shows live messages from truck

    Industry Quest Access

    • Navigate to AWS Industry Quest
    • Find "Manufacturing and Automotive" quest
    • Begin labs in order (they build on each other)

    RockCore Mining Context¶

    Pit Station Enhancement: Supervisors will soon see not just dashboards, but 3D digital representations of haul trucks with live sensor overlays. This enables:

    • Visual anomaly detection (see vibration spikes on the truck model in real-time)
    • Predictive maintenance scheduling (trending on visual twins, not just tables)
    • Fleet-wide digital simulation (run "what-if" scenarios before deploying changes)
    • Training tool (new operators see truck behavior in digital form first)

    Hands-On: Build Your First Digital Twin¶

    Step 1: Create TwinMaker Scene¶

    AWS Console → IoT TwinMaker:

    1. Create new Workspace:
    2. Name: rockcore-mining

    3. Role: Create new service role

    4. Create new Scene:

    5. Name: haul-truck-fleet
    6. 3D Model Source: Matterport Capture or Upload 3D Model (or use bundled models)
    7. Alternative: Create simple primitives (cube for truck body, cylinder for wheels)

    Step 2: Model Truck Entity¶

    In TwinMaker:

    {
  "entityId": "RockCore-Truck-01",
  "entityName": "Haul Truck 01",
  "description": "3D digital twin of haul truck with live sensor telemetry",
  "components": {
    "EngineTemperature": {
      "type": "aws:alarm:temperature",
      "properties": {
        "current_temperature": { "value": 0 },
        "threshold": { "value": 65 }
      }
    },
    "Vibration": {
      "type": "aws:sensor:vibration",
      "properties": {
        "vibration_level": { "value": 0 },
        "anomaly_detected": { "value": false }
      }
    },
    "GasLevel": {
      "type": "aws:sensor:gas",
      "properties": {
        "gas_ppm": { "value": 0 }
      }
    },
    "CabinLock": {
      "type": "aws:switch:lockstate",
      "properties": {
        "locked": { "value": true }
      }
    }
  }
}

    Step 3: Connect Live Data¶

    Option A: Via IoT Analytics (if using from Week 14) - TwinMaker → Data Connectors → Create connector to IoT Analytics channel - Map: temperature field → EngineTemperature.current_temperature - Map: vibration field → Vibration.vibration_level

    Option B: Via SiteWise (if created in Week 14) - TwinMaker → Data Connectors → Create SiteWise connector - Select asset: RockCore-Truck-01 - Auto-map properties to digital twin components

    Option C: Direct IoT Core (advanced) - Use TwinMaker Custom Connector (Lambda-based) - Write Lambda to subscribe to MQTT and push to TwinMaker API

    Step 4: Visualize in 3D Scene¶

    1. In your haul-truck-fleet scene, place the truck 3D model
    2. Select the model → Assign to entity: RockCore-Truck-01
    3. Add overlays showing:
    4. Temperature in top-left corner
    5. Vibration as animated bar on truck body (red if > threshold)
    6. Gas level as gauge

    7. Cabin lock status as icon

    8. Save Scene and open Live View

    9. Verify that as you run your truck, values update in real-time

    Troubleshooting¶

    Q: TwinMaker not available in my region?
    A: TwinMaker is available in select regions (N. Virginia, Frankfurt, Tokyo). Use N. Virginia if your default region lacks it.

    Q: How do I get 3D models of vehicles?
    A: Thingiverse and Sketchfab have free haul truck / mining equipment models. Download as .glb or .obj, then upload to TwinMaker.

    Q: My truck data isn't showing in the digital twin?
    A: Check that IoT Core rule is routing data to SiteWise/Analytics, and that TwinMaker connector is configured correctly. Test with aws iot-data publish to verify message flow.

    Q: Can I use a simple 2D placeholder instead of a 3D model?
    A: Yes — TwinMaker supports 2D scenes too. For now, a simple rectangle labeled "Truck-01" is acceptable; enhance later.
