    Session 06/07: RFID, Access Control & Ultrasonic Distance¶

    Week: 6/7
    Element: ICTIOT502 Element 1: Identify business requirements | ICTIOT502 Element 2: Program IoT device | ICTIOT503 Element 4: Test connectivity and compatibility
    Duration: 3.5 hours
    Phase: Electronics for Programmers

    Note: All electronics sessions can be completed using either C++ (Arduino IDE) or MicroPython on your microcontroller. You may choose your preferred language for all programming tasks, code submissions, and portfolio checkpoints.

    Session Introduction¶

    This combined session covers the Operator Cabin Safety System and the Haul Road Proximity Monitor — two subsystems of the RockCore haul truck. You'll implement RFID card authentication with buzzer-based audio feedback, then add ultrasonic distance sensing to your roof panel for obstacle detection.

    This session completes Assessment 2 (Operator Cabin Safety System).

    Learning Objectives¶

    By the end of this session, you will be able to:

    • Interface an RFID-RC522 module over the SPI protocol
    • Explain the difference between SPI and I²C and when to use each
    • Generate distinct audio tones with a piezo buzzer for access granted / denied feedback
    • Read an ultrasonic distance sensor
    • Design secure access control logic for industrial equipment
    • Complete Assessment 2 with requirements documentation

    Session Structure¶

    1. SPI vs I²C — Choosing the right protocol
    2. RFID Authentication — Card reading and validation over SPI
    3. Buzzer Tones — Audio feedback for access control
    4. Ultrasonic Distance — Collision avoidance
    5. Assessment 2 Submission — Integrated cabin + roof panel system

    RockCore Mining Context¶

    Only authorised operators may start a RockCore haul truck. Your cabin system must reject unauthorised cards with an audible alarm and approve authorised cards with a clear confirmation tone — workers in noisy mine sites rely on sound as much as sight.

    Your roof panel must also detect nearby obstacles to protect the operator and the machine on hazardous haul roads.

    SPI vs I²C — Choosing the Right Protocol¶

    Before wiring the RFID module, it's worth understanding why it uses SPI rather than I²C. You've already used I²C extensively (OLED, MPU6050) — here's how the two protocols compare:

    Feature I²C SPI
    Wires 2 (SDA + SCL) 4 (MOSI, MISO, SCK, SS)
    Addressing 7-bit address per device Separate CS/SS pin per device
    Speed 100 kHz – 400 kHz typical 1–80 MHz typical
    Duplex Half-duplex Full-duplex
    Best for Multiple slow-to-medium devices (sensors, displays) Fast data transfer, one-to-one (RFID, SD cards, displays)
    Wiring complexity Low (shared bus) Higher (one SS pin per device)

    Rule of thumb: - Use I²C when you have several devices to chain on two wires and speed is not critical - Use SPI when you need higher throughput or your device requires full-duplex communication (like the RFID-RC522)

    In this course

    You've used I²C for the OLED (0x3C) and MPU6050 (0x68) — both slow-to-medium sensors sharing two wires. The RFID-RC522 uses SPI because it exchanges data rapidly during card reads and has only one device on the bus.

    Hands-On Tasks¶

    Task 1: RFID-RC522 Card Reading (SPI)¶

    Wiring (SPI protocol):

    RFID Pin ESP32 GPIO
    SDA (SS) GPIO 5
    SCK GPIO 18
    MOSI GPIO 23
    MISO GPIO 19
    RST GPIO 27
    3.3V 3.3V
    GND GND 
    // Week 6/7 Task 1 — RFID Card Reader (SPI)
#include <SPI.h>
#include <MFRC522.h>

#define SS_PIN  5
#define RST_PIN 27

MFRC522 rfid(SS_PIN, RST_PIN);

void setup() {
  Serial.begin(115200);
  SPI.begin();
  rfid.PCD_Init();
  Serial.println("RockCore Access Control: Ready");
  Serial.println("Scan RFID card...");
}

void loop() {
  if (!rfid.PICC_IsNewCardPresent() || !rfid.PICC_ReadCardSerial()) {
    return;
  }

  String cardID = "";
  for (byte i = 0; i < rfid.uid.size; i++) {
    cardID += String(rfid.uid.uidByte[i], HEX);
  }

  Serial.print("Card UID: ");
  Serial.println(cardID);

  rfid.PICC_HaltA();
  delay(1000);
}

    Why SPI here?

    Notice we call SPI.begin() before rfid.PCD_Init(). The RFID module communicates entirely over the SPI bus — contrast this with the OLED and MPU6050 from session ⅚ that use Wire.begin() for I²C.

    Task 2: Buzzer Tones for Access Feedback¶

    A piezo buzzer provides immediate audio feedback in a noisy mine environment. Design distinct tones that are easy to tell apart:

    Wiring: - Buzzer → GPIO 26 - GND → GND

    // Week 6/7 Task 2 — Buzzer Tones
const int BUZZER_PIN = 26;

// Access GRANTED — two rising notes (upbeat, positive)
void toneGranted() {
  tone(BUZZER_PIN, 880, 150);   // A5 — short mid note
  delay(200);
  tone(BUZZER_PIN, 1319, 300);  // E6 — longer high note
  delay(350);
}

// Access DENIED — three short descending buzzes (alarm-like)
void toneDenied() {
  for (int i = 0; i < 3; i++) {
    tone(BUZZER_PIN, 400, 120);  // Low buzz
    delay(180);
  }
}

void setup() {
  pinMode(BUZZER_PIN, OUTPUT);
  Serial.begin(115200);
  Serial.println("Tone test — listen for granted / denied patterns");
}

void loop() {
  Serial.println("Playing GRANTED tone...");
  toneGranted();
  delay(2000);

  Serial.println("Playing DENIED tone...");
  toneDenied();
  delay(2000);
}

    Tone design

    Rising tones feel positive; falling or repeating low tones feel like a warning. Choose frequencies that cut through background noise (400–1400 Hz is easiest to hear over machinery).

    Task 3: RFID Access Control with Buzzer (Assessment 2)¶

    Combine RFID reading with LED indicators and buzzer tones into the complete cabin access system.

    // Week 6/7 Task 3 — Complete Cabin Access System
#include <SPI.h>
#include <MFRC522.h>

#define SS_PIN    5
#define RST_PIN   27
#define GREEN_LED 26   // swap with buzzer pin in your wiring if needed
#define RED_LED   25
#define BUZZER_PIN 33

MFRC522 rfid(SS_PIN, RST_PIN);

// Add your card UIDs here (scan with Task 1 to find them)
String authorisedCards[] = {"a3b2c1d0", "12345678"};
const int numAuthorised = 2;

void toneGranted() {
  tone(BUZZER_PIN, 880, 150);
  delay(200);
  tone(BUZZER_PIN, 1319, 300);
  delay(350);
}

void toneDenied() {
  for (int i = 0; i < 3; i++) {
    tone(BUZZER_PIN, 400, 120);
    delay(180);
  }
}

bool isAuthorised(String cardID) {
  for (int i = 0; i < numAuthorised; i++) {
    if (cardID == authorisedCards[i]) return true;
  }
  return false;
}

void setup() {
  Serial.begin(115200);
  SPI.begin();
  rfid.PCD_Init();
  pinMode(GREEN_LED, OUTPUT);
  pinMode(RED_LED, OUTPUT);
  pinMode(BUZZER_PIN, OUTPUT);
  Serial.println("=== RockCore Cabin Access System ===");
}

void loop() {
  if (!rfid.PICC_IsNewCardPresent() || !rfid.PICC_ReadCardSerial()) {
    return;
  }

  String cardID = "";
  for (byte i = 0; i < rfid.uid.size; i++) {
    cardID += String(rfid.uid.uidByte[i], HEX);
  }

  bool authorised = isAuthorised(cardID);

  Serial.print("Card: ");
  Serial.print(cardID);

  if (authorised) {
    Serial.println(" → ACCESS GRANTED");
    digitalWrite(GREEN_LED, HIGH);
    digitalWrite(RED_LED, LOW);
    toneGranted();
    delay(3000);
    digitalWrite(GREEN_LED, LOW);
  } else {
    Serial.println(" → ACCESS DENIED");
    digitalWrite(GREEN_LED, LOW);
    digitalWrite(RED_LED, HIGH);
    toneDenied();
    delay(3000);
    digitalWrite(RED_LED, LOW);
  }

  rfid.PICC_HaltA();
}

    Task 4: Ultrasonic Distance Sensor (HC-SR04)¶

    Wiring:

    HC-SR04 Pin ESP32 GPIO
    Trig GPIO 12
    Echo GPIO 14
    VCC 5V
    GND GND 
    // Week 6/7 Task 4 — Ultrasonic Distance
const int TRIG_PIN = 12;
const int ECHO_PIN = 14;

float getDistance() {
  digitalWrite(TRIG_PIN, LOW);
  delayMicroseconds(2);
  digitalWrite(TRIG_PIN, HIGH);
  delayMicroseconds(10);
  digitalWrite(TRIG_PIN, LOW);

  long duration = pulseIn(ECHO_PIN, HIGH, 30000);  // 30 ms timeout
  return duration * 0.034 / 2.0;                   // cm
}

void setup() {
  Serial.begin(115200);
  pinMode(TRIG_PIN, OUTPUT);
  pinMode(ECHO_PIN, INPUT);
}

void loop() {
  float dist = getDistance();

  Serial.print("Distance: ");
  Serial.print(dist);
  Serial.println(" cm");

  if (dist > 0 && dist < 50) {
    Serial.println("WARNING: Obstacle within 50 cm!");
  }

  delay(500);
}

    Task 5: Integrated Roof & Cabin System¶

    Bring together the RFID access system and roof panel sensors into one program.

    // Week 6/7 Task 5 — Full Cabin + Roof Panel Integration
#include <SPI.h>
#include <MFRC522.h>

#define SS_PIN     5
#define RST_PIN    27
#define GREEN_LED  26
#define RED_LED    25
#define BUZZER_PIN 33
#define TRIG_PIN   12
#define ECHO_PIN   14

MFRC522 rfid(SS_PIN, RST_PIN);

String authorisedCards[] = {"a3b2c1d0", "12345678"};
const int numAuthorised = 2;

void toneGranted() {
  tone(BUZZER_PIN, 880, 150);  delay(200);
  tone(BUZZER_PIN, 1319, 300); delay(350);
}

void toneDenied() {
  for (int i = 0; i < 3; i++) { tone(BUZZER_PIN, 400, 120); delay(180); }
}

bool isAuthorised(String id) {
  for (int i = 0; i < numAuthorised; i++) {
    if (id == authorisedCards[i]) return true;
  }
  return false;
}

float getDistance() {
  digitalWrite(TRIG_PIN, LOW);  delayMicroseconds(2);
  digitalWrite(TRIG_PIN, HIGH); delayMicroseconds(10);
  digitalWrite(TRIG_PIN, LOW);
  long dur = pulseIn(ECHO_PIN, HIGH, 30000);
  return dur * 0.034 / 2.0;
}

void setup() {
  Serial.begin(115200);
  SPI.begin();
  rfid.PCD_Init();
  pinMode(GREEN_LED, OUTPUT);
  pinMode(RED_LED, OUTPUT);
  pinMode(BUZZER_PIN, OUTPUT);
  pinMode(TRIG_PIN, OUTPUT);
  pinMode(ECHO_PIN, INPUT);
  Serial.println("=== RockCore Cabin + Roof Panel Active ===");
}

void loop() {
  // --- Roof panel sensors ---
  float dist = getDistance();

  if (dist > 0 && dist < 50) {
    Serial.printf("ROOF: Obstacle at %.1f cm\n", dist);
    tone(BUZZER_PIN, 2000, 300);
  }

  // --- Cabin RFID ---
  if (rfid.PICC_IsNewCardPresent() && rfid.PICC_ReadCardSerial()) {
    String cardID = "";
    for (byte i = 0; i < rfid.uid.size; i++) {
      cardID += String(rfid.uid.uidByte[i], HEX);
    }

    if (isAuthorised(cardID)) {
      Serial.println("CABIN: Access GRANTED — " + cardID);
      digitalWrite(GREEN_LED, HIGH); digitalWrite(RED_LED, LOW);
      toneGranted();
      delay(3000);
      digitalWrite(GREEN_LED, LOW);
    } else {
      Serial.println("CABIN: Access DENIED — " + cardID);
      digitalWrite(GREEN_LED, LOW); digitalWrite(RED_LED, HIGH);
      toneDenied();
      delay(3000);
      digitalWrite(RED_LED, LOW);
    }

    rfid.PICC_HaltA();
  }

  delay(500);
}

    Check Your Knowledge¶

    Q1 — SPI vs I²C

    You need to add both an RFID reader and a second OLED to your ESP32. Which protocol does each use, and how does this affect wiring?

    Answer

    The RFID-RC522 uses SPI (requires its own SS/CS pin — GPIO 5). The OLED uses I²C (shares SDA/SCL with any other I²C device, identified by its 0x3C address). You'd wire both protocols simultaneously — SPI bus for RFID, I²C bus for OLED.

    Q2 — Tone Design

    Why not just use the same tone for both access granted and denied?

    Answer

    In a noisy mine environment, visual cues (LEDs) may be missed. Distinct tones provide an additional safety channel. Operators learn to associate the pattern with the outcome without needing to look at the device.

    Q3 — RFID Authorisation

    Your authorised card UIDs are stored in a plain array in code. What is the security risk, and how might you improve it?

    Answer

    Anyone with access to the source code can read the UIDs and add their own card. Improvements: store UIDs in encrypted EEPROM, use a backend server to validate, add a secondary PIN requirement.
