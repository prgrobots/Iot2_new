## Session Introduction

This week focuses on **real-time analytics and data processing** at the cloud layer using **AWS IoT Analytics** for building data pipelines, **SQL queries** for transforming telemetry, and **dashboard creation** for monitoring truck fleet metrics. ## Hands-On Tasks

### Task 1: Create IoT Analytics Channel

**AWS Console → IoT Analytics:**

**Step 1: Create Channel**
- Name: `rockcore_telemetry_channel`
- Retention: 30 days (adjustable based on storage needs)
- Source: Accept messages from IoT Core

**Step 2: Create IoT Rule to Route Data**

In IoT Core → Message routing → Rules, create a rule:

```json
{
  "ruleName": "rockcore_to_analytics",
  "topicRulePayload": {
    "ruleDisabled": false,
    "sql": "SELECT * FROM 'rockcore/trucks/+/telemetry'",
    "actions": [{
      "iotAnalytics": {
        "channelArn": "arn:aws:iotanalytics:region:account:channel/rockcore_telemetry_channel",
        "channelName": "rockcore_telemetry_channel",
        "roleArn": "arn:aws:iam::account:role/IoTAnalyticsRole"
      }
    }]
  }
}
```

**Verify:** Publish test message and check channel metrics in IoT Analytics console.

### Task 2: Create Pipeline for Data Transformation

**Pipeline Purpose:** Clean, validate, and aggregate raw telemetry into a structured dataset.

**Step 1: Create Data Store**
- Name: `rockcore_data_store`
- Retention: 90 days (raw processed data)

**Step 2: Create Pipeline**

In IoT Analytics → Pipelines, create pipeline:

```yaml
Name: rockcore_telemetry_pipeline
Source: rockcore_telemetry_channel

Activities:
  - Channel:
      name: IngestFromChannel
      
  - Datastore:
      name: StoreProcessedData
      datastoreName: rockcore_data_store
```

**Step 3: Add Data Validation (Optional)**

Use attribute filter to validate data:
- Temperature: 0-120°C
- Vibration: 0-10g
- Fuel: 0-100%

### Task 3: Build SQL Query for Real-Time Analytics

**AWS IoT Analytics → Datasets**

Create a dataset that processes data in real-time using SQL:

```sql
-- Real-time truck telemetry aggregation (5-minute buckets)
SELECT 
  truck_id,
  DATE_TRUNC('minute', from_unixtime(timestamp/1000)) as time_bucket,
  ROUND(AVG(temperature), 2) as avg_temperature,
  MAX(temperature) as max_temperature,
  MIN(temperature) as min_temperature,
  ROUND(AVG(vibration), 2) as avg_vibration,
  MAX(vibration) as max_vibration,
  COUNT(*) as sample_count,
  MAX(fuel_level) as fuel_level_latest
FROM rockcore_data_store
WHERE 
  temperature > 0 AND temperature < 120
  AND vibration >= 0 AND vibration <= 10
GROUP BY 
  truck_id,
  DATE_TRUNC('minute', from_unixtime(timestamp/1000))
ORDER BY 
  time_bucket DESC,
  truck_id
```

**Schedule:** Every 5 minutes (minimum interval for real-time insights)

**Retention:** 7 days for quick-access dashboard, archive older data to S3

### Task 4: Create Dataset for Historical Analysis

**30-Day Trend Analysis:**

```sql
-- Daily aggregation for trend detection
SELECT 
  truck_id,
  DATE_TRUNC('day', from_unixtime(timestamp/1000)) as date,
  ROUND(AVG(temperature), 2) as daily_avg_temp,
  ROUND(STDDEV(temperature), 2) as temp_variance,
  ROUND(AVG(vibration), 2) as daily_avg_vibration,
  ROUND(STDDEV(vibration), 2) as vibration_variance,
  COUNT(*) as total_samples
FROM rockcore_data_store
WHERE 
  from_unixtime(timestamp/1000) > current_timestamp - interval '30' day
GROUP BY 
  truck_id,
  DATE_TRUNC('day', from_unixtime(timestamp/1000))
ORDER BY 
  truck_id,
  date DESC
```

**Use Case:** Identify trucks with increasing vibration trend (predictive maintenance signal)

### Task 5: Export Data for Dashboard

**Create Dataset for QuickSight Integration:**

```sql
-- Fleet summary for dashboard KPIs
SELECT 
  COUNT(DISTINCT truck_id) as total_trucks,
  ROUND(AVG(temperature), 1) as fleet_avg_temp,
  MAX(temperature) as fleet_max_temp,
  ROUND(AVG(vibration), 2) as fleet_avg_vibration,
  COUNT(*) as total_messages,
  ROUND(COUNT(*) / COUNT(DISTINCT DATE_TRUNC('hour', from_unixtime(timestamp/1000))), 0) as messages_per_hour,
  MAX(from_unixtime(timestamp/1000)) as latest_update
FROM rockcore_data_store
WHERE from_unixtime(timestamp/1000) > current_timestamp - interval '1' hour
```

**Schedule:** Every 1 minute (real-time KPI updates)

### Task 6: Store Aggregated Data in S3 (Optional Advanced)

**For long-term storage and ML training:**

```sql
-- Export daily summaries to S3 for archival and ML
CREATE EXTERNAL TABLE rockcore_daily_archive (
  truck_id STRING,

  sample_count INT
```

## Check Your Knowledge

!!! question "Q1 – IoT Analytics vs. Lambda Rules"
    When should you use IoT Analytics instead of Lambda functions triggered by IoT Rules?
    ??? tip "Answer"
        Use IoT Analytics for: 1) Complex data transformations, 2) Historical queries over large datasets, 3) Time-series aggregations (GROUP BY time), 4) Scheduled batch processing. Use Lambda for: 1) Simple real-time actions, 2) Immediate alerts, 3) Device-specific logic.

!!! question "Q2 – Data Retention Strategy"
    You have 20 trucks publishing 1 message/sec (86.4M messages/day). Store raw data 30 days = ~2.6B records. How reduce costs?
    ??? tip "Answer"
        1) Aggregate to 1-min averages in pipeline (1440 records/truck/day instead of 86,400), 2) Archive raw data to S3 Glacier after 7 days, 3) Partition datasets by truck_id and date for fast queries, 4) Use retention policies to auto-delete old data.

!!! question "Q3 – SQL Aggregation Functions"
    Why use ROUND(AVG(vibration), 2) instead of just AVG(vibration)?
    ??? tip "Answer"
        ROUND() reduces data precision for display (avoiding floating-point noise), reduces storage size when stored, and makes dashboards more readable. Statistical analysis still works correctly with rounded averages.
