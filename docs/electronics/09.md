## MCU Roles in This Session

!!! info "Two Pico W Boards: Truck MCU + Display MCU"
    Session 9 requires **2 Pico W boards** because the OLED display and GY-521 vibration sensor cannot run on the same board.

    **Architecture:**
    - **Board 1 (Master MCU):** Runs all truck sensors (thermistor, RFID, servo, RGB, buzzer, **GY-521**). Publishes sensor data to AWS IoT Core and to Board 2 via inter-board communication.
    - **Board 2 (Slave MCU):** Runs the **OLED display**. Receives sensor data from Board 1 via I²C or serial communication and displays live truck status.

    **What this means practically:**

    - Inter-board communication: Configure I²C (if boards are close) or serial (UART via USB bridge if separate). Master (Board 1) sends sensor readings; Slave (Board 2) displays them on the OLED.
    - The filtering algorithm you wrote in Session 8 runs on Master (Board 1). Slave (Board 2) is a pure display client.
    - Both boards run the Arduino framework (C++) with identical build toolchain.

## Hands-On Tasks

### Task 1: Board 1 Truck Controller - Sensor Aggregation

```cpp
// Week 9 Task 1 - Board 1: Truck Controller
#include <Wire.h>
#include <MPU6050.h>
#include <Adafruit_SSD1306.h>

// Thermistor (NTC 10kΩ) — same circuit as Assessment 1
#define THERMISTOR_PIN 33
const float BETA    = 3950;
const float R_FIXED = 10000.0;

MPU6050 mpu;
Adafruit_SSD1306 display(128, 64, &Wire, -1);

struct TruckData {
  float temperature;
  int gasLevel;
  float vibration;
  int rainLevel;
  int lightLevel;
  float distance;
  bool accessGranted;
  String status;
};

TruckData truck;

void setup() {
  Serial.begin(115200);
  Wire.begin();
  mpu.begin(MPU6050_SCALE_2000DPS, MPU6050_RANGE_2G);
  display.begin(SSD1306_SWITCHCAPVCC, 0x3C);
  
  Serial.println("=== RockCore Fleet Truck HC-01 ===");
  truck.status = "INITIALIZING";
}

void updateSensors() {
  // Thermistor temperature read (Steinhart-Hart)
  int raw = analogRead(THERMISTOR_PIN);
  if (raw > 0 && raw < 4095) {
    float resistance = R_FIXED * raw / (4095.0 - raw);
    float tempK = 1.0 / (1.0 / 298.15 + (1.0 / BETA) * log(resistance / 10000.0));
    truck.temperature = tempK - 273.15;
  }
  
  Vector accel = mpu.readNormalizeAccel();
  truck.vibration = sqrt(accel.XAxis * accel.XAxis + 
                         accel.YAxis * accel.YAxis + 
                         accel.ZAxis * accel.ZAxis);
  
  truck.lightLevel = analogRead(36);
  
  // Update status based on all sensors
  if (truck.temperature > 80) {
    truck.status = "FAULT";
  } else if (truck.temperature > 60 || truck.vibration > 1.5) {
    truck.status = "WARNING";
  } else {
    truck.status = "OPERATIONAL";
  }
}

void updateDisplay() {
  display.clearDisplay();
  display.setCursor(0,0);
  display.setTextSize(1);
  display.print("Truck: HC-01");
  display.println(truck.status);
  display.println("----------------");
  display.print("Temp: ");
  display.print(truck.temperature, 1);
  display.println("C");
  display.print("Gas: ");
  display.println(truck.gasLevel);
  display.print("Vibe: ");
  display.println(truck.vibration, 2);
  display.print("Rain: ");
  display.println(truck.rainLevel < 2000 ? "YES" : "NO");
  display.display();
}

void loop() {
  updateSensors();
  updateDisplay();
  
  // Serial logging (JSON format for AWS later)
  Serial.print("{\"truck\":\"HC-01\",\"temp\":");
  Serial.print(truck.temperature);
  Serial.print(",\"gas\":");
  Serial.print(truck.gasLevel);
  Serial.print(",\"vibe\":");
  Serial.print(truck.vibration, 3);
  Serial.print(",\"status\":\"");
  Serial.print(truck.status);
  Serial.println("\"}");
  
  delay(2000);
}
```

// Add to master controller loop()

void testFaultConditions() {
  // Check for critical faults
  bool criticalFault = false;
  String faultMessage = "";
  
  if (truck.temperature <= -998) {  // readTemp() returns -999 on ADC out-of-range
    criticalFault = true;
    faultMessage = "SENSOR FAULT: Thermistor open circuit";
  }
  
  if (truck.temperature > 80) {
    criticalFault = true;
    faultMessage = "CRITICAL: Engine overheat";
  }
  
  if (truck.gasLevel > 500) {
    criticalFault = true;
    faultMessage = "CRITICAL: Gas leak detected";
  }
  
  if (truck.vibration > 2.0) {
    criticalFault = true;
    faultMessage = "CRITICAL: Severe vibration";
  }
  
  if (criticalFault) {
    digitalWrite(RED_LED, HIGH);
    tone(BUZZER, 2000, 1000);
    Serial.print("FAULT: ");
    Serial.println(faultMessage);
    truck.status = "SHUTDOWN";
  }
}

// Week 9 Task 3 - Fault Recovery
void checkSensorHealth() {
  // Thermistor health check (ADC returns 0 or 4095 if open/short circuit)
  int raw = analogRead(THERMISTOR_PIN);
  if (raw == 0 || raw == 4095) {
    Serial.println("WARNING: Thermistor fault - using last known value");
    // Use cached value or safe default
  }
  
  // I²C device health check
  Wire.beginTransmission(0x3C);  // OLED
  if (Wire.endTransmission() != 0) {
    Serial.println("WARNING: OLED offline - continuing without display");
  }
  
  Wire.beginTransmission(0x68);  // GY-521

    Serial.println("WARNING: GY-521 offline - vibration monitoring disabled");

}

## Check Your Knowledge

!!! question "Q1 – I²C Bus Load"
    Your system has 2 I²C devices (OLED on Board 2, GY-521 on Board 1). What's the maximum you can add?
    ??? tip "Answer"
        I²C supports 127 devices (7-bit addressing), but practical limit is ~10-20 due to capacitance and speed degradation. Each device adds capacitance to SDA/SCL lines.

!!! question "Q2 – System-Level Fault"
    If the master Pico W crashes, what happens to all subsystems?
    ??? tip "Answer"
        Complete system failure – single point of failure. Industrial systems use redundant controllers or watchdog timers that reset the MCU on hang.

!!! question "Q3 – Power Budget"
    Calculate total current draw: Pico W (100mA) + OLED (20mA) + 3 sensors (50mA each) + servo (500mA). Can you power this from USB (500mA)?
    ??? tip "Answer"
        Total = 770mA. USB can't supply this safely. Need external 5V power supply (2A minimum) or separate servo power.
